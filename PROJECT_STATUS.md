# 项目状态记录 (PROJECT_STATUS)

## 版本历史

### v1.3 - 2026-02-21

**新增功能：**

1. **Web 监控看板**：
   - 新增 `web/index.html`，提供可视化的策略状态监控界面。
   - 实时展示：
     - **账户概览**：总资产、可用金额、持仓市值等核心指标。
     - **当前持仓**：持股明细、成本价、现价估算及盈亏比例。
     - **黑名单监控**：当前被策略排除的风险标的。
   - 数据源直接读取本地 JSON 文件 (`持股数据.json`, `账户数据.json`, `黑名单.json`)。

2. **便捷启动脚本**：
   - 新增 `start_web.py`，一键启动本地 HTTP 服务并自动打开浏览器访问监控页面。
   - 解决了静态页面无法跨目录读取 JSON 数据的问题。

### v1.2.1 - 2026-02-21

**修复内容：**

1. **配置项修复**：
   - 修复 `qmt_trader_ths` 模块运行时的 `KeyError: '软件授权码'` 错误。
   - 在 `策略配置.json` 中新增 `"软件授权码"` 字段（默认值 "免费版"）。

2. **错误处理增强**：
   - `法玛全天候绝对动量指数风控策略.py`：
     - `connact()` 方法增加对配置项缺失的捕获与提示。
     - 主程序启动时若连接交易系统失败，将直接报错退出，避免无效运行。

### v1.2 - 2026-02-21

**更新内容：**

1. **日志系统实装**：
   - 引入 Python `logging` 模块，替代原有的 `print` 输出。
   - 实现按日期自动轮转的日志文件存储：`log/YYYY/MM/YYYY-MM-DD.log`。
   - 自动创建日志目录结构，确保日志文件有序存放。

2. **核心模块日志改造**：
   - `法玛全天候绝对动量指数风控策略.py`：主策略逻辑全面接入日志系统，记录交易信号、持仓检查、风控触发等关键信息。
   - `trader_tool/base_func.py`：基础工具函数增加日志记录。
   - `trader_tool/seed_trader_info.py`：消息通知模块（邮件/钉钉/微信）增加发送状态日志。

### v1.1 - 2026-02-21

**更新内容：**

1. **股票池机制重构**：

   - 支持从通达信板块文件 (`.blk`) 动态加载股票池，替代原有的 JSON 静态列表。
   - 新增 `blk/` 目录用于存放股票池文件（如 `ZZYQ.blk`）。
   - 修改 `策略配置.json`，增加 `"股票池文件"` 字段，移除硬编码的 `"股票池"` 和 `"股票池名称"`。
2. **代码逻辑优化**：

   - `trader_tool/base_func.py`：新增 `read_blk_file` 函数，自动解析 `.blk` 文件并转换代码格式（0->SZ, 1->SH）。
   - `法玛全天候绝对动量指数风控策略.py`：
     - `get_trader_stock()`：适配文件读取逻辑，优先从 `blk/` 目录加载。
     - `get_rank()`：移除对旧版股票名称映射的依赖，直接使用股票代码进行排名。
3. **数据存储优化**（v1.0 后续）：

   - 废弃 Excel (`.xlsx`) 数据存储，全面转向 JSON 格式 (`持股数据.json`, `账户数据.json`)。
   - 增强了跨平台兼容性（使用 `os.path.join` 处理路径）。

### v1.0 - 初始版本

- 基础策略框架：法玛全天候绝对动量 + 指数风控。
- 支持 QMT 交易接口。
- 包含基础的动量计算、风控逻辑和交易执行模块。
